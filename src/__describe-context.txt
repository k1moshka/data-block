1. contextHandler - это обработчик который определяется на стадии определения блока


2. contextFactory - это фабрика, которая создает провайдеров контекста


3. contextProvider - это билдер инстанса контекста


4. contextInstance - это непосредственная функция 
    которая запускает нужный обработчик из иерархии контекстов


1. Describe
    - register context handler (just wrap it for nested namespaces)

2. Instancing
    - create context factory for creating context runners for rendering 
        -> contextFactory

3. Rendering
    - create runner context for rendering session -> contextInstance
    
4. Post-rendering
    - run contextInstance with newValue, oldValue and path
