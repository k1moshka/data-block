1. handlerHandler - это обработчик который определяется на стадии определения блока


2. handlerFactory - это фабрика, которая создает провайдеров контекста


3. handlerProvider - это билдер инстанса контекста


4. handlerInstance - это непосредственная функция
    которая запускает нужный обработчик из иерархии контекстов


1. Describe
    - register handler handler (just wrap it for nested namespaces)

2. Instancing
    - create handler factory for creating handler runners for rendering
        -> handlerFactory

3. Rendering
    - create runner handler for rendering session -> handlerInstance

4. Post-rendering
    - run handlerInstance with newValue, oldValue and path
